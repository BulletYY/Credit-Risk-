def historical_method(zwroty:np.array,quantile=0.99):
    """
    Implementacyjnie sa 2 podejscia , albo bierzemy minus przed zwrotami i liczymy jak dla prawo stronnego rozkladu
    albo liczymy bez minusa ale wtedy liczymy quantyl jako 1- signif_level 
    Ja wybralem podejscie 1. (oba sa z ksiazki Hull'a)

    """
    var_value = np.quantile(zwroty,1-quantile)
    
    es_value = np.mean(zwroty[zwroty <= var_value])
    
    
    return np.round(var_value,4), np.round(es_value,4)

def monte_carlo_method(zwroty:np.array, quantile=0.99, num_of_simulations=100000):
    
    mean_dist = np.mean(zwroty)
    std_dist = np.std(zwroty,ddof=1)
    
    zwroty =norm.rvs(mean_dist,std_dist, size=num_of_simulations)
    
    var_value = np.quantile(zwroty,1-quantile)
    
    es_value = np.mean(zwroty[zwroty <= var_value])
    
    
    return np.round(var_value,4), np.round(es_value,4)

def christoffersen_independence(violations):

    n00 = n01 = n10 = n11 = 0
    for i in range(1, violations.shape[0]):
        if violations[i-1] == 0 and violations[i] == 0: 
            n00 += 1
        elif violations[i-1] == 0 and violations[i] == 1: 
            n01 += 1
        elif violations[i-1] == 1 and violations[i] == 0: 
            n10 += 1
        elif violations[i-1] == 1 and violations[i] == 1: 
            n11 += 1

    if (n00 + n01 == 0) or (n10 + n11 == 0): # nieokreslony div by 0 
        return np.nan

    pi0 = n01 / (n00 + n01)
    pi1 = n11 / (n10 + n11)
    pi  = (n01 + n11) / (n00 + n01 + n10 + n11)


    return -2 * np.log( (1 - pi)**(n00 + n10) * pi**(n01 + n11) / ((1 - pi0)**n00 * pi0**n01 *(1 - pi1)**n10 * pi1**n11)  )

Podsumowanie obydwu metod

 

- Traffic light system metoda historyczna przez wiekzosc czasu jest w green zone ani razu nie weszla w red zone najczesciej wystepuje w green zone zwykle 0-4 naruszen max 8 okolo 7% razy.

- W Metodzie monte carlo okolo 16% razy wpadlismy w strefe czerwona, okolo 21% w stefie zoltej i okolo tylko 63% razy w stefie zielonej , co jest zdecydowanie gorszym wynikiem niz metoda historyczna

- Kupiec tylko 7% razy zostala odrzucona hipoteza H0 w metodzie historycznej gdzie powinien okolo odrzucic 12-13 razy hipoteze H0, zatem ta metoda niedoszacowywuje ryzyka, ponieważ zbyt często odrzuciliśmy hipoteze H0.

- Test kupca w metodzie monte carlo odrzucil hipoteze H0 około 23 % razy co od razu wskazuje ze zdecydowanie gorzej niedoszacowywuje ryzyko niż metoda historyczna.

- Pod wzgledem metody Christoffersona obydwie metody daja wiarygodne oszacowania poniewaz nie daja ani razu odrzucenia hipotezy H0.

- Brak odrzucenia w żadnym z okiem hipotezy H0 w teście Christoffersona wskazuje na brak klastrowania breachów varu. Natomiast sam holt piszę, że sam test bardzo mocno zalęzy od częstości z jaką występują po sobie breache varu, jeśli są one rzadkie, test na małą moc, co widac po wykresach u góry.

- Dodatkowo nie defniuje on co jeśli nie ma żadnych po sobie breachów varu.

- Podsumowanie

- Metoda monte carlo nie daje wiargodnych oszacowan ponieważ nie doszacowywuje ryzyka, jest ona zdecydowanie gorsza niż metoda historyczna (oczywiście biorąc pod uwage rozkład normalny,inne rozkłady np t zdecydowanie mogłyby polepszyć wyniki).

- Metoda historyczna tez w pelni nie daje wiargydodnego oszacowania var, poniewaz test kupca odrzuca czesciej hipoteze H0 niz wynosi poziom istotnosci, natomiast nie aż tak mocno niedoszacowywuje ryzyka jak metoda MC.

- Teoretycznie możnabyłoby połączyć metode historyczną + wagi + heterskedastyczność co zdecydowanie polepszyłoby estymowanie ryzyka.
